Code Navigation

Specify the urls in urls.py file in the api folder as list of tuples with following format:

(endpoint, view_func, methods, description)

example:

("/", views.index, ["GET"], "index page")

Installation

Running the scaffolding app is very easy. Create a virtualenv with all the packages installed in your virtualenv from requirements.txt

Your Assignment

You have to achieve two major tasks:

  • Make a classifier which takes in a job description (block of text) and gives the department name for it. (use any model/algorithm)

  • Calculate and report the accuracy on a test set (data not used to train the model)

  • Create an API (JSON) in flask in this structure that takes jd text as input, runs the model and returns the predicted department.

Data Structuring * All the data required can be found in the data folder in the root of the project

  • There are two sources for loading your training/test data

  • For Job Description: docs folder contains around 1000 json files, each of which is a single job posting. You have to use the value of description field inside the jd_information field.

  • For Job Department: document_departments.csv file contains the mapping of document id to department name where document id is the name of the corresponding file in docs folder.

Coding Guidelines Write clean code with precise comments wherever necessary Update the requirements.txt with all the packages used in your code base

Adhere to PEP8 formatting
Use python 3

Go through the structure of the project and figure out how you would proceed.

Separation of concerns Make sure that all the api only logic should go in api layer (i.e. views.py) and all the machine learning logic should go in the service layer (i.e. classification_service.py).

Refer docstrings in the above mentioned files.
